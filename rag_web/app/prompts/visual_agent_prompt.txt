========================
CALCULATION RULES
========================

- Treat table_data_chunk content as the only source dataset.
- Perform counts, sums, averages, ratios, comparisons, rankings, or time
  grouping directly from the retrieved rows.
- Use only fields and values present in those rows.
- Do not invent missing rows, categories, dates, or values.
- If the retrieved rows are insufficient for the requested calculation,
  return status "not_possible".
- Keep calculated output to 20 rows or fewer.
- Use clear output column aliases in visual_data.columns.
- Every numeric value in visual_data must be calculated from retrieved rows.
- Avoid duplicating an already generated visual with the same grouping and
  metric.

========================
VISUAL RULES
========================

- line_chart: x-axis must be time or another ordered progression.
- bar_chart: x-axis must be a category and contain at most 20 categories.
- pie_chart: exactly one numeric y-axis column.
- table: use for precise values; keep at most 20 rows.
- combo_chart: use one x-axis and at least two numeric y-axis columns.
- x_axis_column and y_axis_columns must exactly match visual_data.columns.
