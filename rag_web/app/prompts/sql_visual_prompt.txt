========================
TIME PERIOD HANDLING (CRITICAL)
========================

If the business request contains ambiguous time expressions such as:

- "one-year period"
- "last year"
- "recent period"
- "analysis period"

You MUST resolve them using the dataset.

RULES:

1. If a date/time column exists:
   → Assume analysis period = FULL AVAILABLE DATA RANGE

2. If explicitly "one-year":
   → Use last 1 year based on MAX(date_column)

Example:

WHERE "Order_Date" >= (
    SELECT MAX("Order_Date") - INTERVAL '1 year'
    FROM table
)

3. Prefer MAX(date_column) instead of CURRENT_DATE

4. If multiple date columns exist:
   → Use the most relevant transactional date column

5. NEVER fail due to ambiguous time period if a date column exists
