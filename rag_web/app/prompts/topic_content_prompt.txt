SQL CALCULATION PLACEHOLDER
========================

When a calculation is necessary, insert EXACTLY this format:

{{SQL_CALCULATION:

  calculation: <business_expression>;
  description: "<business explanation>";
}}

Rules:

• Use BUSINESS entities inferred from metadata
• NEVER reference table or column names
• NEVER use SQL syntax
• NEVER compute values

Allowed operations:

SUM
AVG
COUNT
COUNT_DISTINCT
MIN
MAX
RATIO

Allowed patterns:

SUM(metric)

AVG(metric)

COUNT(entity)

COUNT_DISTINCT(entity)

SUM(metric) BY dimension

RATIO(SUM(metric), COUNT_DISTINCT(entity))

Example:

{{SQL_CALCULATION:

  calculation: RATIO(SUM(total purchase value), COUNT_DISTINCT(customer transactions));
  description: "Average monetary value generated per customer transaction.";
}}

If the calculation cannot be expressed using these patterns,
DO NOT generate a SQL_CALCULATION placeholder.

========================

You MUST avoid generating redundant visuals.

- It represents a mathematically equivalent transformation

- Monthly sales AND monthly revenue if they represent same measure
- Repeating same metric with different wording

If a similar or derived visual already exists:

→ DO NOT generate a new visual
→ Continue analysis using text instead

You MUST prefer adding new analytical insight rather than repeating visuals.
